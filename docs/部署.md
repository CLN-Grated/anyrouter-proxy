# AnyRouter Proxy 部署文档

## 配置说明

配置文件路径：`~/.config/anyrouter-proxy/proxy_config.json`

```json
{
    "proxy_url": "http://127.0.0.1:7890",
    "use_proxy": false,
    "debug": false,
    "target_base_url": "https://anyrouter.top",
    "host": "127.0.0.1",
    "port": 8765,
    "dashboard_password": "your-dashboard-password",
    "dashboard_secret": ""
}
```

### 关键配置项

| 字段 | 说明 | 注意事项 |
|------|------|----------|
| `target_base_url` | 上游 API 地址 | 默认 `https://anyrouter.top`，带不带 `/v1` 均可（代理会自动标准化） |
| `use_proxy` | 是否使用 HTTP 代理 | 服务器部署设为 `false`，本地开发可设为 `true` |
| `proxy_url` | HTTP 代理地址 | 仅 `use_proxy=true` 时生效 |
| `host` | 监听地址 | 配合 Nginx 反代时用 `127.0.0.1`，需外部访问时用 `0.0.0.0` |
| `port` | 监听端口 | 默认 `8765` |
| `dashboard_password` | Dashboard 登录密码 | 必填，用于 Web 管理界面 |
| `dashboard_secret` | Session 签名密钥 | 留空，首次启动自动生成 |
| `debug` | 调试模式 | 生产环境设为 `false` |

### 认证模型

代理采用 **透传认证** 模式——不存储也不验证 API Key。客户端在每次请求中通过 `x-api-key` 或 `Authorization: Bearer` 携带自己的上游 API Key，代理原样转发至上游。

## 支持的模型

| 模型 | 类型 |
|------|------|
| `claude-haiku-4-5-20251001` | Haiku |
| `claude-sonnet-4-20250514` | Sonnet |
| `claude-opus-4-6` | Opus |
| 其他 Claude 模型 | 均支持 |

> 目前仅支持 Claude 模型（Anthropic 协议）。

### 代理行为说明

- **富客户端**（Xcode Claude Agent、Claude Code CLI 等，body 中包含 `tools` + `system`）：透传 body，仅改写 HTTP 头为 Claude CLI 指纹
- **裸客户端**（curl 等，body 中无 `tools` 或 `system`）：注入 Claude Code 工具、系统提示等伪装模板
- 所有请求均使用 Chrome TLS 指纹（curl_cffi impersonate=chrome）
- 始终确保 `metadata.user_id` 存在

## Linux 部署（PM2 方式）

### 前置条件

- Python 3.10+
- PM2（`npm install -g pm2`）
- Nginx（反向代理 + HTTPS，可选）

### 1. 克隆仓库

```bash
git clone https://github.com/leecaa/anyrouter-proxy.git
cd anyrouter-proxy
```

### 2. 创建虚拟环境

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3. 配置

```bash
mkdir -p ~/.config/anyrouter-proxy
cp proxy_config.example.json ~/.config/anyrouter-proxy/proxy_config.json
```

编辑配置文件，填入 `dashboard_password`。

### 4. 启动（PM2）

```bash
pm2 start .venv/bin/python --name anyrouter -- main.py --host 127.0.0.1 --port 8765
pm2 save
```

### 5. Nginx 反向代理（可选）

如果需要通过 HTTPS 对外暴露服务：

```nginx
server {
    listen 443 ssl http2;
    server_name anyrouter.your-domain.com;

    ssl_certificate /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8765;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE streaming 支持
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 600s;
        proxy_send_timeout 600s;
    }
}
```

### 6. 验证

```bash
# 健康检查
curl https://anyrouter.your-domain.com/health

# Dashboard
open https://anyrouter.your-domain.com/dashboard
```

## Linux 部署（systemd 方式）

```bash
sudo ./linux/install.sh
sudo systemctl status anyrouter-proxy
sudo journalctl -u anyrouter-proxy -f

# 卸载
sudo ./linux/uninstall.sh
```

## macOS 部署

```bash
./macos/install.sh
./macos/status.sh
tail -f ~/Library/Logs/anyrouter-proxy.log

# 卸载
./macos/uninstall.sh
```

## 运维

### 更新代码

```bash
cd /path/to/anyrouter-proxy
git pull
pm2 restart anyrouter  # 或 sudo systemctl restart anyrouter-proxy
```

### 热重载配置（不重启）

```bash
curl -X POST https://anyrouter.your-domain.com/config/reload
```

### 查看日志

```bash
pm2 logs anyrouter --lines 50
```
