# AnyRouter Proxy 交接说明

## 项目概览

- 本项目是代理桥接服务，将客户端请求伪装为 Claude CLI 流量转发到 AnyRouter 上游
- 目前仅支持 Claude 模型（Anthropic 协议）
- 运行方式：Python + FastAPI + uvicorn，默认监听 `127.0.0.1:8765`
- 主要用途：为 Xcode Claude Agent、Claude Code CLI、OpenCode、CherryStudio 等客户端提供 Anthropic 兼容接口

## 目录结构

| 文件/目录 | 说明 |
|----------|------|
| `main.py` | 核心代理服务：请求转发、指纹伪装、客户端检测 |
| `claude_code_tools.json` | Claude Code 工具注入模板（裸客户端用） |
| `claude_code_system.json` | Claude Code 系统提示注入模板（裸客户端用） |

| `model_tester.py` | 模型连通性测试 |
| `proxy_config.example.json` | 配置样例 |
| `docs/` | 文档目录 |
| `macos/` | macOS LaunchAgent 部署脚本 |
| `linux/` | Linux systemd 部署脚本 |

## 核心行为

### 请求处理逻辑

1. **HTTP 头伪装**：所有请求的 HTTP 头均改写为 Claude CLI 指纹（User-Agent、X-Stainless-*、anthropic-beta 等）
2. **TLS 指纹**：使用 curl_cffi 的 Chrome impersonate 模式，绕过 WAF 检测
3. **Body 处理**：
   - **富客户端**（body 含 `tools` + `system`，如 Xcode、Claude Code）→ 透传 body，不做修改
   - **裸客户端**（body 无 `tools` 或 `system`，如 curl）→ 注入 Claude Code 伪装模板
4. **metadata.user_id**：始终确保存在，格式为 `user_{hash}_account__session_{uuid}`
5. **API Key 透传**：代理不存储 Key，从客户端请求中提取并转发到上游

### anthropic-beta 标志

当前完整的 beta 标志列表（`_ANTHROPIC_BETA_FULL`）：

## 配置

默认路径：`~/.config/anyrouter-proxy/proxy_config.json`（可通过 `ANYROUTER_PROXY_CONFIG` 环境变量覆盖）

详细配置说明参见 [部署文档](部署.md)。

## 服务端点

| 端点 | 方法 | 说明 |
|-----|------|------|
| `/health` | GET | 健康检查 |
| `/config` | GET | 读取配置（已脱敏，需认证） |
| `/config/reload` | POST | 热重载配置（需认证） |
| `/v1/{path}` | ALL | 代理转发（Anthropic 兼容路径） |
| `/dashboard` | GET | Web 管理界面（需认证） |
| `/api/test-all` | POST | 批量测试所有模型（需认证） |
| `/api/test/{model}` | POST | 测试单个模型（需认证） |

## 常见问题排查

1. **`/health` 无响应**：检查进程是否运行（`pm2 status` 或 `systemctl status`）
2. **`All connection attempts failed`**：上游代理未启动，检查 `use_proxy` 和 `proxy_url` 配置
3. **`1m 上下文` 错误**：检查 `_ANTHROPIC_BETA_FULL` 是否包含 `context-1m-2025-08-07`
4. **配置修改不生效**：调用 `POST /config/reload` 或重启进程
5. **Xcode 显示错误模型身份**：更新到最新版本（旧版会覆盖客户端的 system prompt）
